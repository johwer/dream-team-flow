# Worktree Port Setup Guide

When running multiple worktrees (one per ticket), each needs its own ports so they don't collide. This guide explains what you need in your repo to make it work.

## How It Works

```
Main repo (port 3000)     ← always running
Worktree PLRS-1234 (port 3142)  ← ticket 1
Worktree PLRS-1235 (port 3143)  ← ticket 2
```

DTF's `allocate-ports.sh` derives unique ports from the ticket number. No manual config needed.

## What You Need in Your Repo

### 1. Vite Config — Read Port from Environment

Your `vite.config.mts` needs to read the port from an env variable with a fallback:

```ts
// vite.config.mts
import { defineConfig, loadEnv } from "vite";

export default defineConfig(({ mode }) => {
  const env = loadEnv(mode, process.cwd(), "VITE_");

  return {
    server: {
      port: Number(env.VITE_DEV_PORT || "3000"),
    },
    preview: {
      port: Number(env.VITE_DEV_PORT || "3000"),
    },
    // If you proxy API calls, read ports from env too:
    // proxy: {
    //   "/api/service-a": {
    //     target: `http://localhost:${env.VITE_SERVICE_A_PORT || "5001"}`,
    //     changeOrigin: true,
    //   },
    //   "/api/service-b": {
    //     target: `http://localhost:${env.VITE_SERVICE_B_PORT || "5002"}`,
    //     changeOrigin: true,
    //   },
    // },
  };
});
```

**Key point:** The fallback (`"3000"`) means non-worktree users are unaffected. They never set the env var, so they get the default port.

### 2. Docker Compose — Use Variable Ports

If you run backend services in Docker, use env variables for port mappings:

```yaml
# docker-compose.yml
services:
  my-api:
    build: .
    ports:
      - "${MY_API_PORT:-5001}:5001"  # Default 5001, overridden in worktrees
    environment:
      ASPNETCORE_URLS: http://+:5001  # Internal port stays fixed
```

**The pattern:** External port is variable (`${MY_API_PORT:-5001}`), internal port is fixed (`:5001`). Worktrees change the left side, app code doesn't change.

### 3. Docker Compose for Worktrees (Optional)

If you want worktree services to connect to the main stack's infrastructure (database, Redis, etc.), create a separate compose file:

```yaml
# docker-compose.worktree.yml
services:
  my-api-wt:  # "-wt" suffix avoids DNS collision with main stack
    build:
      context: ${WORKTREE_BUILD_CONTEXT:-.}
      dockerfile: path/to/Dockerfile
    ports:
      - "${MY_API_PORT:-6001}:5001"
    environment:
      ASPNETCORE_URLS: http://+:5001
      ConnectionStrings__DefaultConnection: Host=postgres;Database=mydb;Username=postgres;Password=postgres
    networks:
      - main-network

networks:
  main-network:
    external: true
    name: ${MAIN_NETWORK:-myproject_default}  # Your main stack's Docker network name
```

**Important:** The `-wt` suffix on service names prevents Docker from round-robin routing between main and worktree containers.

### 4. `.gitignore` Entries

Add these to your `.gitignore`:

```gitignore
# Worktree port allocation (generated by DTF)
.env
apps/web/.env.local
apps/web/vite.config.worktree.mts

# Worktree artifacts
.dream-team/
__screenshots__/
```

### 5. Company Config — Define Your Services

In your `company-config.json`, list the services that need port mapping:

```json
{
  "services": {
    "ServiceA": "YourServiceA",
    "ServiceB": "YourServiceB"
  }
}
```

DTF uses this during `dtf install` to replace generic names with your real service names in all config files.

## How `allocate-ports.sh` Works

```
Ticket PLRS-1234
  → Extract number: 1234
  → Slot: 1234 mod 99 + 1 = 47
  → Vite port: 3100 + 47 = 3147
  → API base: 10000 + 47 * 100 = 14700
  → Service A: 14701
  → Service B: 14702
  → etc.
```

- Checks for collisions with other active worktrees
- Writes `.env` at worktree root with all port variables
- Writes `apps/web/.env.local` with Vite-specific ports
- Non-worktree users always get default ports (3000, 5001, 5002, etc.)

## Adapting `allocate-ports.sh` for Your Project

The script is generic except for service names. Edit the port mapping section (lines ~82-91) to match your services:

```bash
# Calculate ports from slot
VITE_PORT=$((3100 + 10#$SLOT))
API_BASE=$((10000 + 10#$SLOT * 100))

# Change these to match your services:
SERVICE_A_PORT=$((API_BASE + 1))
SERVICE_B_PORT=$((API_BASE + 2))
SERVICE_C_PORT=$((API_BASE + 3))
```

And update the `.env` output section to write your service port variables.

## Adapting `docker-compose.worktree.yml` for Your Project

1. Copy the template and rename services to match yours
2. Update Dockerfile paths
3. Update database names and connection strings
4. Update the network name to match your main docker-compose network
5. Add/remove services as needed

## Quick Test

After setting up:

```bash
# 1. Start main stack
docker compose up -d

# 2. Create a worktree
git worktree add ~/Documents/TEST-001 -b TEST-001

# 3. Allocate ports
bash ~/.claude/scripts/allocate-ports.sh TEST-001

# 4. Start frontend in worktree
cd ~/Documents/TEST-001/apps/web
npm start  # Should use port from .env.local, not 3000

# 5. Verify
curl -s http://localhost:3000 > /dev/null && echo "Main: OK"
curl -s http://localhost:31XX > /dev/null && echo "Worktree: OK"

# 6. Clean up
cd ~/Documents/YourProject
git worktree remove ~/Documents/TEST-001
```

## Files Summary

| File | In Repo | Generated | Purpose |
|------|---------|-----------|---------|
| `vite.config.mts` | Yes (edit once) | No | Read port from env with fallback |
| `docker-compose.yml` | Yes (edit once) | No | Variable external ports |
| `docker-compose.worktree.yml` | Yes (template) | No | Worktree services on main network |
| `.env` | No (gitignored) | Yes (per worktree) | Docker port variables |
| `apps/web/.env.local` | No (gitignored) | Yes (per worktree) | Vite port + proxy targets |
| `.gitignore` | Yes (edit once) | No | Ignore generated files |
| `allocate-ports.sh` | No (in `~/.claude/`) | DTF provides | Port allocation script |
